* ZeroClaw Setup

After deploying the NixOS configuration, ZeroClaw needs to be onboarded interactively on the host.

** 1. Populate the Anthropic API key secret

#+begin_src bash
cd secrets
agenix -e zeroclaw-anthropic-api-key.age
#+end_src

** 3. Run onboarding on the host

SSH into claw and run onboarding. The =zeroclaw= command automatically runs as the zeroclaw user via a wrapper. This creates the config at =/data/zeroclaw/.zeroclaw/config.toml= and sets up channels (Telegram, WhatsApp Web, etc.).

#+begin_src bash
ssh claw.stfl.dev
zeroclaw onboard --interactive
#+end_src

** 4. Pair WhatsApp Web

After onboarding, pair WhatsApp Web via the QR code flow:

#+begin_src bash
zeroclaw channel pair whatsapp
#+end_src

** 5. Start the daemon

#+begin_src bash
sudo systemctl start zeroclaw
sudo systemctl status zeroclaw
journalctl -u zeroclaw -f
#+end_src

* Cloudflare Tunnel Setup

Follow the [[https://developers.cloudflare.com/cloudflare-one/connections/connect-networks/get-started/create-local-tunnel/][Cloudflare local tunnel setup guide]].

The tunnel is managed from a host with =modules/cloudflared.nix= (e.g. kondor),
which provides the =cloudflared= CLI and the origin certificate (=cert.pem=). The
target host (claw-pve) only needs the tunnel credentials to run the tunnel.

** 2. Create a tunnel

On the management host.

#+begin_src bash
cloudflared tunnel create <NAME>
#+end_src

This outputs a tunnel UUID and a credentials JSON file (=~/.cloudflared/<UUID>.json=).

- Encrypt the credentials JSON into =secrets/cloudflared-tunnel-credentials.age=:

#+begin_src bash
cd secrets && agenix -e cloudflared-tunnel-credentials.age
#+end_src

- Set the tunnel UUID in =services.cloudflared.tunnels.<UUID>= in =hosts/claw-pve/default.nix=

** 3. Route DNS

For each ingress hostname, create a CNAME pointing to the tunnel:

#+begin_src bash
cloudflared tunnel route dns <NAME> n8n.stfl.dev
cloudflared tunnel route dns <NAME> monica.stfl.dev
#+end_src

** 4. Deploy to target

#+begin_src bash
nixos-rebuild --target-host claw.stfl.dev --sudo switch --flake ".#claw-pve"
#+end_src

** Management

List and inspect tunnels from the management host:

#+begin_src bash
cloudflared tunnel list
cloudflared tunnel info <NAME-or-UUID>
#+end_src
